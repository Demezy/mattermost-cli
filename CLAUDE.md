# mattermost-cli (mm)

A CLI tool for interacting with Mattermost servers via the REST API.

## Tech Stack

- **Runtime:** Bun (not Node.js)
- **Language:** TypeScript, strict mode
- **CLI framework:** citty (unjs)
- **Test runner:** `bun:test`

## Commands

See `justfile` for all available recipes (`just --list`). Common ones:

```bash
just run <args>                      # Run the CLI
just run-test <args>                 # Run CLI with .env.test loaded
just test                            # Run all tests
just lint                            # Type check
just docker-setup                    # Full Docker test env setup
```

To load env vars from a file manually, use `bun --env-file=<path>`:

```bash
bun --env-file=.env.test bin/mm.ts   # Loads .env.test before running
```

The `.env.test` file (generated by `docker/init.sh`) points at the local
Docker test instance and is the default for development.

## Naming Conventions

- **Files:** kebab-case (`url-parser.ts`, `global-args.ts`)
- **Functions:** camelCase, verb-first (`createClient`, `resolveUsers`)
- **Types/Interfaces:** PascalCase, noun-phrase (`ConnectionConfig`, `MattermostClient`)
- **Constants:** camelCase in code, UPPER_SNAKE for env var names
- **Exports:** Named only â€” no default exports (exception: citty command files
  use `export default defineCommand(...)` for lazy import compatibility)

## Patterns

- **No classes** unless genuinely necessary. Use factory functions returning
  plain objects.
- **No global state.** Dependencies flow as function arguments.
- **Named exports** from `index.ts` files as module public APIs.
- **Output to stdout** (data), **logs to stderr** (progress, errors).
- **Typed errors.** `MattermostApiError` carries `statusCode` and
  `mattermostErrorId`. No raw stack traces in production output.
- `process.exit()` only in `bin/mm.ts`, never in library code.

## Environment Variables

| Variable       | Required | Description                          |
|----------------|----------|--------------------------------------|
| `MM_URL`       | Yes      | Mattermost server base URL           |
| `MM_TOKEN`     | Yes      | Personal access token                |
| `MM_TEAM_ID`   | No       | Team ID (required for `mm channels`) |
| `MM_READ_ONLY` | No       | Set `true` to block write operations |
